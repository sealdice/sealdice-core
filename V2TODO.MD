# V2 API 迁移清单

本清单用于跟踪后端 API 从旧版 Echo v1 路由 `/sd-api` 迁移到基于 Huma 的 v2 路由 `/sd-api/v2` 的进度。

范围说明：

- 只统计后端 API，不参考前端或生成客户端。
- `[x]` 表示已经存在 v2 Huma 路由，并且大体已有可用实现。
- `[ ]` 表示尚未迁移，或者只有不完整的初版/占位实现。
- 有些 v1 功能后续可能会在 v2 中移除；在明确记录移除决策之前，仍保持未完成状态。

## 基础信息 / 鉴权

- [x] 健康检查：公开检查服务存活、测试模式、初始化状态
- [x] 运行时概览：迁移 `baseInfo`，按应用信息、版本信息、运行状态、环境信息分组返回
- [x] 鉴权探针（`hello`）：由需要鉴权的 v2 接口替代，不再单独迁移
- [x] 登录
- [x] 登录盐获取（`signin/salt`）
- [x] 安全检查
- [ ] 强制停止
- [x] 后台日志 WebSocket 推送：使用 Melody 建立日志订阅，连接后发送历史快照并实时推送新增日志
